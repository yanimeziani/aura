# Contributing to Aura

Thank you for your interest in contributing.

## Getting started

- **Setup and concepts:** See [docs/ONBOARDING.md](docs/ONBOARDING.md) for environment setup, modes, and daily commands.
- **Architecture:** See [docs/HIERARCHY.md](docs/HIERARCHY.md) for high-level structure.

## Before you contribute

- **Keep secrets out of the repo.** Use environment variables or the vault; never commit `.env`, `aura-vault.json`, or real API keys. See [SECURITY.md](SECURITY.md).
- **Subprojects** (e.g. `dragun-app/`, `aura-landing-next/`) may have their own CONTRIBUTING or AGENTS.md; follow those when changing that area.

## How to contribute

1. Open an issue or discussion for larger changes.
2. Fork the repo and create a branch from the default branch.
3. Make your changes; run any existing tests and linters for the area you touch.
4. Submit a pull request with a clear description of what changed and why.

By contributing, you agree that your contributions will be licensed under the same license as the project (see [LICENSE](LICENSE)).
