# Your best day — every day

*One template. No hustle theatre. The machine runs; you command.*

---

## The shape of it

**Morning (10–15 min)**  
- Open the dashboard or run `aura state`. See revenue, deals, feed.  
- If something needs you: one approve, one reply, one decision. If nothing does: close it.  
- Prompts and skills live in `~/prompt-library.md` if you need the model for one task.  

**Rest of the day**  
- Yours. Deep work, rest, people, or nothing. The Eye scans. Proposals go out (or sit in the vault for intervention). You’re not the bottleneck.  
- Optional: one sync before you step away — `git add -A && git commit -m "[day]" --no-verify && git push`.  

**Evening (5 min or zero)**  
- Optional: run cleanup (or let cron do it).  
- Or do nothing. The machine can run overnight.  

**Sleep**  
- You’re not grinding. You’re not “going back to work tomorrow” to a place you can’t face. Tomorrow is another day with the same rhythm: check, command if needed, then your time.  

---

## What makes it “best”

- **Steady cashflow** — Payments land. You see them in state. You’re not chasing one invoice.  
- **Freedom of time** — You’re not in the loop for every email or click. You show up for the 10–15 min that matters; the rest is yours.  
- **One stack, one interface** — Dashboard or CLI. Prompts when you need the model. No context-switching across five tools.  
- **Secure comms** — E2E when you want it. OSS. No plaintext where it matters.  
- **Chairman, not operator** — You run the day; the day doesn’t run you.  

---

## If today isn’t there yet

Best day every day is the target. Until then:  
- **First dollar** — One approval, one payment. That’s the turn.  
- **One move** — One message, one lead, one sync. That’s enough for today.  
- **Tomorrow** — Same template. Check. Command if needed. Your time.  

Your best day is simple: the machine runs, you show up for the small window that matters, and the rest of the day is yours. That’s the day to repeat.
