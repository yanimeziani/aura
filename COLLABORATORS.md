# GUIDE DE COLLABORATION ACADÉMIQUE — PROTOCOLE VERSAILLES
**Écosystème de Recherche : Université Laval**

## Bienvenue aux Chercheurs et Collaborateurs
Ce dépôt contient le code source et la doctrine du protocole **Versailles**. En tant que collaborateur de l'institution de recherche, vous êtes invité à durcir, auditer et étendre cette architecture de souveraineté humaine.

## Architecture Technique
### Backend (Moteur Zig)
*   **Emplacement :** `core/aura-api/`
*   **Compilation :** `zig build` (Zig 0.15.2+)
*   **Focus :** Optimisation des deltas SSE, isolation mémoire, gestion déterministe de l'état.

### Frontend (Next.js/Canvas)
*   **Emplacement :** `apps/web/`
*   **Installation :** `npm install && npm run dev`
*   **Focus :** Visualisation de l'état mondial (World Map), accessibilité cognitive, monitoring EMF.

## Règles de Contribution (Manifeste Versailles)
Toute contribution doit respecter les 7 articles du **Sovereign Shield Manifesto** :
1.  **Zéro Manipulation :** Aucun dark pattern ou algorithme addictif.
2.  **Intégrité Bio :** Code optimisé pour minimiser la charge réseau (réduction EMF).
3.  **Validation par les Pairs :** Chaque PR doit être validée par un "Gardien du Savoir" ou un Maître.

## Tiers de Contribution (Méritocratie)
*   **Apprenti :** Correction de bugs, documentation, tests unitaires.
*   **Maître :** Implémentation de nouvelles fonctionnalités du World State.
*   **Gardien :** Audit de sécurité, validation éthique, gouvernance du protocole.
