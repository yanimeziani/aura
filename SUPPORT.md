# Support

## Use GitHub Issues For

- reproducible bugs
- documentation gaps
- protocol/spec inconsistencies
- onboarding friction
- feature proposals aligned with the architecture

## Use Security Reporting For

- vulnerabilities
- trust-boundary bypasses
- secret exposure
- exploit chains

See [SECURITY.md](/root/SECURITY.md).

## Before Opening an Issue

- read [README.md](/root/README.md)
- check [docs/SEED.md](/root/docs/SEED.md)
- check [docs/ARCHITECTURE_DISTILL.md](/root/docs/ARCHITECTURE_DISTILL.md)
- include concrete steps, environment details, and observed behavior
